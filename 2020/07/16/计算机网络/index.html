2. 物理层

物理层的功能：

主要功能：提供透明的比特流传输
注意：
1）封装好的数据以“0、1”比特流的形式进行传递，从一个地方搬到另一个地方；
2）物理层上的传输，从不关心比特流里面携带的信息，只关心比特流的正确搬运。

四大特性：机械特性、电气特性、功能特性、规程特性

物理层上数据的传输：信号

信号：数据的电气或电磁表现，分为模拟信号和数字信号

信号在信道/传输介质上传输
物理带宽：传输过程中振幅不会明显衰减的频率范围，单位：赫兹
数字带宽：单位时间内流经的信息总量

奈奎斯特定理：在无噪声信道中，当物理带宽为 B Hz，信号离散等级为V级，那么该信道能提供的最大传输速率（数字带宽）：
最大传输速率 = 2Blog₂V（bps）
任意一个信号通过一个物理带宽为 B 的低通滤波器只要进行没秒 2B 次的采样就可完全重构出被滤掉的信号，任何高于2B次的采样都毫无意义。

香农定理：在噪声信道中，如果物理带宽为 B Hz。信噪比为 S/N，那么最大的传输速率（数字带宽）：
最大传输速率 = Blog₂(1 + S/N)（bps）
很多情况下噪声用分贝（DB）表示：
分贝值 = 10log10（S/N）（dB)

复用技术

1.1）频分多路复用FDM
在干线起点，信道的频谱被分成若干段（子带），每个用户占据一段来传输自己的信号；到了干线的终点，每个子带的信号，被单独分离出来给各个用户。
正交FDM：Orthogonal FDM
没有保护带，且子带之间相互重叠，同样的干线可以承载更多的用户

1.2）波分多路复用WDM（Wavelength Division Multiplexing）
本质跟FDM一样，在光纤上复用信号；
当相邻波长间隔非常接近，子信道的数目非常大，WDM变成了DWDM（Dense，密集波分多路复用）

2）时分多路复用TDM（Time Division Multiplexing）
在时间上共享信道：将时间划分为非常短的时间片
统计时分多路复用技STDM（Statistic TDM），动态分配信道

3）码分多路复用技术CDMA（Code Division Multiple Access）
一种扩展频谱技术。
VDMA允许每个站利用整个频段发送信号，而且没有任何时间限制。
TDM—-许多人按顺序交谈
FDM—-不同的人按照不同的语调同时进行交谈
CDMA—-每对交谈者使用不同的语言
CDMA的关键在于能够取出需要的信号同时拒绝所有其它的信号并把这些信号当做噪声。CDMA中，每个比特时间被细分为m个更短的时间间隔，这更短的时间间隔被称为码片，每个比特被分为64或128

纠1位错需要的冗余位跟数据位的关系是

纠1位错的海明码的基本方法是怎样的？
发送方：根据校验集合填充校验码
接收方：根据校验集合判定校验位是否出错，出错的位编号累加到累加器上，所有的校验位都检查完成后，通过读取累加器的值来确定码字中出错的那一位的编号。

6. 传输层

传输实体
网络层负责把数据从源机送达到目的机（Host to Host）
传输层数据送达到具体的应用进程（End to End）

数据段（TPDU，传输层协议数据单元）
TPDU（Transport Protocol Data Unit）是从传输实体发到对端传输实体的信息。有的教材叫TPDU为数据段（segment）
TPDUs作为数据（载荷）被封装在分组（packet）中，由网络层传输交换；
分组作为数据（载荷）被封装在帧（frames）中，由数据链路层传输交换；

传输层主要有两个常用的协议：
用户数据协议UDP（User Datagram Protocol）
传输控制协议TCP（Transmission Control Protocol）

传输层提供高效、可靠的数据传输服务

UDP是传输层上的一个轻量级协议，提供高效的端到端的数据段传输
UDP数据段包括头部和载荷两部分，头部有4个字段，共8个字节
UDP不提供数据传输的可靠保证

端点就是套接字Socket
（IP，Port）
通信五元组：
Source IP、Source port
TCP/UDP
Destination IP、Deationation port

通信模型：通信三元组
源端点、协议、目的端点

TCP是一个复杂的传输层协议，负责将TCP数据段可靠地送达目的端

相比UDP段头，TCP段头非常复杂，包含14个字段
源端口、目的端口（通信五元组中的两个）
序列号：为每个字节编号
确认号：ack控制位置位才有效
六个控制位：用于连接建立、拆除、异常处理等
窗口尺寸：用于流控，以免收方被数据淹没

采用 三次握手 建立连接：
一方（server）被动地等待一个进来的连接请求
另一方（client）通过发送连接请求，设置一些参数
服务器方回发确认应答
应答到达请求方，请求方最后确认，连接建立

1）SYN（SEQ = x），SYN = 1, ACK = 0

2）SYN（SEQ = y，ACK = x + 1），SYN = 1, ACK = 1

3）（SEQ = x + 1，ACK = y + 1），SYN =0, ACK = 1

拒绝服务攻击DoS

TCP数据段传送之前，一定要建立TCP连接
通过三次握手建立TCP连接，三次的控制号见上
三次握手建立连接是一个同步的过程，交换初始序列号，保证后续的每一个字节的可靠传输

​ 任何一方没有数据要传送的时候，都可以发送一个FIN置位了的TCP数据段
​ 当FIN被确认的时候，该方向的连接被关闭
​ 当双向连接都关闭了的时候，连接释放

DR（Disconnect Require）

四次握手 正常释放连接：
1）主机A发送释放请求DR，启动定时器
2）主机B回送确认ACK，到达主机A时，A –> B连接释放
3）主机B发送释放请求DR，启动定时器
4）主机A回送确认ACK，到达主机B时，B –> A连接释放

TCP是全双工的，连接的释放必须是双向的
FIN = 1
半开放连接必须杀掉
定时器的作用很重要，超时将释放连接

TCP传输的是全双工的字节流

TCP适配双方的数据流量的方法是
Window size（窗口尺寸，段头里的 WIN字段）

TCP还需要效率：
发方优化：Nagel算法（延迟发送）
收方优化：Clark解决方案（傻瓜窗口综合症）

接受者窗口、拥塞窗口、阈值

慢启动算法（Slow Start）

TCP拥塞控制遵循分组守恒定律

两种因素引发拥塞：
接收方处理不过来，用窗口尺寸来度量
通信子网中出现拥塞，用拥塞窗口（CWND）来度量

处理拥塞的具体方法是使用窗口尺寸和拥塞窗口中小的那个来发送

CWND通过慢启动方法尝试而来（通过阈值调节CWND尝试的粒度）

TCP中有很多定时器，管理定时器是一件重要的事情
（重传定时器、持续定时器（解决死锁）、保活定时器等）
肯定确认定时器，用来保证数据段的可靠传输
持续定时器用来解除因更新窗口丢失引发的死锁

TCP跟UDP有极大的不同，是一个重量级的传输层协议

性能 TCP UDP

传输延迟来源 不确定 网络延迟
