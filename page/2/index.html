Lu小白的进阶之路：数据结构

1. 绪论

1 - 计算

计算模型 = 计算机 = 信息处理工具
所谓算法，即特定计算模型下，旨在解决特定问题的指令序列。
事实：程序 != 算法； 死循环 ~ 有穷性
好算法：正确、健壮、可读、效率（速度尽可能快、存储空间尽可能少）
（算法 + 数据结构）x 效率 = 计算能力

2 - 计算模型

度量，算法分析
问题实例的规模，往往是决定计算成本的主要因素

Turing Machine
Random Access Machine

3 - 大O记号

时间复杂度O()**，
高效解、有效解、难解
多项式复杂度与指数复杂度是算法有效与否的分水岭
2-Subset is **NP-complete，就目前的计算模型而言，不存在可在多项式时间内回答此问题的算法。

4 - 算法分析

两个主要任务 = 正确性（不变性 x 单调性） + 复杂度
级数：
算数级数：与末项平方同阶；
幂方级数：比幂次高出一阶；
几何级数（a>1)：与末项同阶；
收敛级数：O(1)
循环：
（一般是fou循环嵌套，可类比成坐标轴面积···）

封底估算（Back-Of-The-Envelope Calculation)

5 - 迭代与递归

迭代乃人工，递归方神通，但递归效率未必高。
1）减而治之：为求解一个大规模的问题，可以 将其划分为两个子问题：其一平凡，另一规模缩减，分别求解子问题，由子问题的解，得到原问题的解。
递归跟踪：检查每个递归实例累计所需时间，其总和即算法时间；
递推方程

2）分而治之：为求解一个大规模的问题，可以将其划分为若干（通常两个）子问题，规模大体相当，分别求解子问题，由子问题的解，得到原问题的解。
二分递归：（数组求和）int mid = (low + high) >> 1; 右移一位相当于除以2
二分递归：（Max2）

6 - 动态规划

利用 递归（work 、right）找出算法本质 + 迭代（fast）的形式= 实现动态规划
fib(n) = fib(n-1) + fib(n-2)
自顶向下递归，自底而上迭代

最长公共子序列LCS
对于序列A[0, n]和B[0, m]，LCS(A, B)无非三种情况
0）若n = -1 或 m = -1，则取作空序列（””）//递归基
1）若A[n] = ‘X’ = B[m]，则取作LCS(A[0, N), B[0, m)) + ‘X’ //减而治之
2）A[n] ≠ B[m]，则在LCS(A[0, n], B[0, m)) 与LCS(A[0, n), B[0, m]) 中取更长者 //分而治之

2.向量Vector

1 - 接口与实现

序列Sequence：向量Vector、 列表List

抽象数据类型 = 数据模型 + 定义在该模型上的一组操作
数据结构 = 基于某种特定语言，实现ADT的一整套算法
数据结构涉及到：具体实现、多种实现、内部的表示与实现、完整的算法、与复杂度密切相关、要考虑数据的具体存储机制。

向量ADT（抽象数据类型）：
i）从数组到向量，向量是数组的抽象与泛化，有一组元素按线性次序封装而成；
i）各元素与[0, n)内的秩（rank）一一对应；
i）元素的类型不限于基本类型；
i）操作、管理维护更加简化、统一与安全；
i）可更为便捷地参与复杂数据结构的定制与实现。

操作接口 size() \ sort() \ find() \ search() \ ···
C\C++ 构造与析构

2 - 可扩充向量

静态空间管理：
开辟内部数组 _elem[] 并使用一段地址连续的物理空间
_capacity：总容量
_size：当前的实际规模n
上溢overflow
下溢underflow

动态空间管理：
在即将发生上溢时，适当地扩大内部数组的容量
扩容策略：（扩容的时间成本不同）
递增式扩容、加倍式扩容

平均分析 与 分摊分析
